## ⭐ Part 0. 교육 과정 안내 (1교시 도입 15분)

> **이 섹션이 1회차의 출발점**. 교육생분이 *"오늘 끝나면 나는 무엇을 할 수 있게 되는가"* 를 15분 안에 명확히 가져가게 한다. 슬라이드 6장.

### 0.1 시작 전 가벼운 자가 진단 4문항 (1분)

> 강사가 교육생분에게 묻고, 교육생분은 *손 들기*로 답한다. 슬라이드 한 장에 4문항 모두 표시.

1. 오늘 아침까지 **ChatGPT 또는 Gemini를 한 번이라도 켜 본 분?**
2. AI에게 뭔가 시킬 때, *"어떻게 물어야 잘 답할까"* 를 **5초 이상 고민해 본 분?**
3. AI 답변이 *마음에 안 들어서* 그냥 **창을 닫아본 분?**
4. **본인 업무에 AI를 매일 쓰고 있는 분?**

> 4번에 손 든 사람이 0~1명이라면 — **이 교육은 바로 여러분을 위해 만들어졌습니다**.

### 0.2 본 교육의 큰 그림 — 3회차 9시간이면 도대체 뭐가 달라지나? (3분)

| 회차 | 일자 | 주제 | 회차 끝나는 시점에 내 손에 쥐어지는 것 |
|------|------|------|--------------------------------|
| **1회차 (오늘)** | 5/26 (화) | LLM 원리 + **MD 프롬프트** | 업무 맥락 정의서 + 자동화 후보 3개 |
| 2회차 | 6/2 (화) | 출력 형식 다양화 (Mermaid) + 도구 컨셉 확정 | 업무 흐름도 + 도구 컨셉 1건 |
| 3회차 | 6/9 (화) | 바이브 코딩 + 도구 완성 | **본인만의 HTML 도구 v1** |

> 9시간 뒤, 교육생분 한 분 한 분의 손에 **본인 업무용 도구 한 개**가 쥐어진다. 그 도구의 *씨앗*이 바로 오늘 만들어진다.

### 0.3 본 교육이 **하지 않는 일** (1분)

- ❌ **코딩** — 교육생분이 키보드로 코드를 짤 일은 *없습니다*. 3회차에 AI에게 *시키기만* 합니다.
- ❌ **새 솔루션 도입 결정** — *"○○ AI 도구 사야 할까요?"* 는 본 교육 범위가 아닙니다.

> ✅ **우리가 하는 일**: 이미 손에 쥔 ChatGPT / Gemini를 *매일 매일* 쓸 수 있는 사람으로 만들기.

### 0.4 본 교육에서 사용하는 도구 — 딱 4개 (2분)

| 도구 | 용도 | 비용 |
|------|------|------|
| **ChatGPT** | 대화·프롬프트 실행 | Plus 권장 (Free도 가능) |
| **Gemini** | 대체 가능한 다른 대화 도구 | Advanced 권장 |
| **메모장 또는 워드패드** | MD 파일 만들기 (윈도우 기본 제공) | 0원 |
| **markdownlivepreview.com** | MD가 어떻게 보이는지 미리보기 | 0원 |

> 모두 **이미 가지고 계시거나 무료로 쓸 수 있는 것**. 새 프로그램 설치 없음. **카카오톡으로 친구한테 사진 보내는 것보다 쉬운 도구들**입니다.

### 0.5 ⭐ 1회차 목표 — "오늘 끝나면 이걸 할 수 있게 된다" 4가지 (3분)

> 교육생분이 *집에 들고 가는 것* 4가지.

**① AI가 왜 그런 답을 하는지 *설명할 수 있다***
> 친구가 *"ChatGPT 써봤는데 별 거 없던데?"* 라고 하면, *"맥락을 안 줬으니까 그렇지"* 라고 한 마디 해 줄 수 있는 수준.

**② AI에게 줄 글을 *마크다운으로 정리할 수 있다***
> 메모장에 `# 제목`, `- 목록`, `**굵게**` 다섯 가지만 쓸 줄 알면 됨. 5분이면 배움.

**③ 본인 업무용 *"MD 프롬프트 파일" 하나의 뼈대*를 잡을 수 있다** ← **오늘의 핵심**
> 본인 PC에 `회의록_정리.md` 같은 파일 1개. 매월 회의록을 정리할 때 *그 파일 하나만 복사*하면 끝나는 상태.

**④ 본인 업무에서 자동화할 *후보 3개*를 골라낼 수 있다**
> "이 3개 중에 하나를 2회차에 도구로 만들 거예요" 라고 강사에게 말할 수 있는 상태.

#### 🎯 강사 전용 정답/힌트 (교육생분에게는 비공개)

> ⚠️ **이 박스는 강사용 — 슬라이드 발표자 모드의 노트와 동일 내용**

**▶ 정답**: **해태(獬豸)** — 광화문 앞에 서 있는 한국 전통 신화 속 *수호 동물* 석상.

- 별칭: *해치* / *시비를 가리는 정의의 동물* / *서울시 캐릭터 모티브*
- 한자 표기: 獬豸 (해할 *해* / 짐승 *치*) — 두 글자

**▶ 왜 어려운가**

- 한국인이 *형상*은 알지만 *"해태"* 라는 명칭은 잘 안 떠올림
- 제과 회사 이름("해태제과")으로만 인식하는 경우 다수
- 오답 흐름: *광화문 앞* → *사자상* → *호랑이상* → *돼지상* → … *해태*

**▶ 정답 유도 키워드** (교육생분이 이 단어를 쓰면 AI가 즉시 맞힘)

- *"광화문 앞"*, *"조선시대 석상"*, *"왕궁 앞 수호 동물"*
- *"외뿔이 하나 있는"*, *"비늘로 덮인 사자"*, *"정의를 상징"*

**▶ 게임 종료 후 정리 멘트 (그대로 읽어도 OK)**

> *"같은 사진인데, '광화문 앞' 한 단어만 더했더니 AI가 바로 맞혔죠?"*
> *"이게 '맥락(context)'입니다. 오늘 첫 번째 단어를 외우세요 — 맥락."*
> *"AI는 사진을 본 게 아니에요. 여러분이 쓴 글의 *다음 단어*를 점친 거예요."*

**▶ 응답이 느릴 때 보조 힌트 (단계별)**

1. *"왕궁 앞에 있는 동물상입니다"*
2. *"한자로 두 글자, 첫 글자가 '풀 해(解)'와 발음이 같아요"*
3. *"제과 회사 이름으로 더 유명한 그것 맞습니다"*

### 1.1 핵심 메시지 — *"AI는 다음 단어를 찍는 점쟁이"*

> AI는 *의미를 이해해서 답하는 것*이 아닙니다.
> **여러분이 쓴 글의 *다음에 올 가장 그럴듯한 단어*를 찍는 기계**입니다.
> 그래서 — **앞에 뭘 써 주느냐가 모든 것을 결정합니다.**

### 1.2 빈칸 채우기 — 직장인 일상 버전

> "내일 회의 시간을 ___"

| AI가 가장 가능성 있다고 본 다음 단어 | 확률 |
|------------------------------|---:|
| 잡아주세요 | 38% |
| 조정해주세요 | 27% |
| 알려주세요 | 18% |
| 확인해주세요 | 12% |
| 늦춰주세요 | 5% |

→ AI는 **가장 확률 높은** 단어를 고를 뿐. *"잡아주세요"* 가 가장 흔하게 쓰이니까 가장 위에 옴.

**라이브 시연**: ChatGPT에 *"내일 회의 시간을"* 만 보내고 결과 확인.

> 💡 *교육생분에게 한 마디*: "여기서 *맥락*을 한 줄만 더 주면 답이 어떻게 바뀔까요?"

### 1.3 맥락 한 줄로 답이 바뀐다

> "거래처 부장님께 보낼 메일을 쓰는 중입니다. 내일 회의 시간을 ___"

| 다음 단어 | 확률 |
|----------|---:|
| **말씀드리려 합니다** | 41% |
| 안내드리고자 합니다 | 28% |
| 공지드립니다 | 17% |
| 알려드립니다 | 10% |

**같은 빈칸** → 앞에 *"거래처 부장님께 메일"* 한 줄 추가 → **반말체가 아닌 격식체**로 답이 바뀜.

> 🪄 **마법이 아니라 *확률*** 입니다. AI는 *"부장님께 메일"* 이라는 단어들 다음에 *"말씀드리려"* 가 흔하게 등장한다는 통계를 외우고 있을 뿐.

### 1.4 역할극 — 회의록 정리 시나리오 (8분)

**❌ 나쁜 프롬프트** — 이게 평소 교육생분이 하는 일

```
어제 회의록 정리 좀 해줘

오전 10시 시작
김 부장: 다음달 행사 예산이 부족함
박 차장: 그래서 어떻게 하실 건지
김 부장: 일단 본사에 추가 요청
이 과장: 예산은 다음주 화요일까지 확정 필요
박 차장: 알겠음
```

→ AI가 내놓는 결과: *어색한 표 + 행동 항목 누락 + 담당자 모호*.
→ 교육생분의 반응: *"음... 별로네"* → 창 닫음 → **다시 손으로 정리**.

**✅ 좋은 프롬프트** — 오늘 끝나면 교육생분이 *이렇게* 쓸 수 있게 된다

```markdown
[역할]
당신은 (주)멋진엔지니어링 관리부 회의록 담당자입니다.

[작업]
아래 회의 메모를 회사 표준 회의록 양식으로 정리해 주세요.

[판단 기준]
- 결정 사항과 의견을 구분
- 행동 항목은 반드시 담당자와 기한을 포함
- 발언자 이름은 "○○ 부장", "○○ 과장" 형태로 통일
- 책임 소재가 불명확한 항목은 **[확인필요]** 표기

[출력 형식]
## 회의록

| 항목 | 내용 |
|------|------|
| 일시 | (메모에서 추출) |
| 참석자 | (메모에서 추출) |

### 결정 사항
- (목록)

### 검토 사항 (담당 / 기한)
- 항목: ○○○ / 2026-06-XX

### 다음 회의 안건
- (목록)

[메모]
(여기에 회의 메모 본문 붙여넣기)
```

→ AI가 내놓는 결과: **사내 결재용 회의록에 그대로 붙여넣을 수 있는 수준**.

**라이브 시연 노트**:
- 한 번에 점프 ❌. 단계별로 시연: **(1) 메모만 → (2) 역할 추가 → (3) 판단 기준 추가 → (4) 출력 형식 추가**
- 매 단계마다 *결과가 어떻게 좋아지는지* 교육생분이 *육안으로* 보게 함
- 마지막 한 마디: *"이 좋은 프롬프트 전체를 메모장에 저장해 두면, 다음 회의 때 메모만 갈아 끼우면 됩니다. 3교시에 정식으로 배웁니다."*

### 1.5 비교: 같은 AI, 다른 결과

|       | 나쁜 프롬프트 | 좋은 프롬프트 |
|-------|--------------|--------------|
| 역할 부여 | ❌ 없음 | ✅ 회의록 담당자 |
| 출력 형식 | ❌ AI가 알아서 (대개 어색) | ✅ 사내 표준 양식 |
| 판단 기준 | ❌ 누락 | ✅ 명시 |
| 결과 활용성 | "다시 손으로 고쳐야 함" | "복사해서 결재판에 붙임" |
| 다음 달 재사용 | 매번 새로 씀 | **파일 복사 한 번** |

### 1.6 ⏱ 실습 ① — 본인 업무에 적용 (15분)

**진행**

1. 본인 업무 중 *AI에게 시키고 싶은 작은 작업* 1개 떠올리기
   - 예: 메일 초안, 공문 한 문단, 안내 문자, 회의 안건 정리, 자료 한 줄 요약
2. **맥락 없이** 한 줄로 시켜보기 → 결과 받기
3. **역할 + 판단 기준 + 출력 형식** 추가해서 다시 시키기 → 결과 받기
4. 두 결과를 *나란히* 놓고 비교

**기록 (워크시트)**

| 칸 | 적기 |
|---|------|
| 어떤 작업? | (한 줄) |
| 나쁜 결과의 *모자란 점* | (한 문장) |
| 좋은 결과의 *나아진 점* | (한 문장) |

→ 마지막 3분 한두 분 공유.

## 2교시 — Part 2. 핵심 원리 2: 의미공간 (25분)

### 2.1 핵심 메시지 — *"AI는 단어를 별자리로 본다"*

> 엑셀의 `VLOOKUP`은 글자가 *완벽히 일치*해야 매칭합니다.
> AI는 다릅니다. AI는 단어를 **별자리처럼** 봅니다.
> 비슷한 의미는 *가까이*, 다른 의미는 *멀리* — 글자 하나가 달라도 *같은 무리*로 인식.

### 2.2 별자리 비유 — 5초 만에 이해하기

> "휴가"라는 단어 주변에 어떤 단어들이 모여 있을까요?

```
                   📚 휴직
                       
   🌴 연차       😴 휴식
                       
        🤒 병가       
              ☂ 휴가 (중심)
        🧓 경조
                       
   🚪 반차         📅 연가
                       
              🕐 외출
```

→ AI는 *"휴가 / 연차 / 반차 / 연가 / 휴직 / 병가 / 경조 / 외출"* 을 **같은 동네 단어**로 인식.
→ VLOOKUP은 *"연차"* 와 *"연가"* 가 *다른 글자* 라서 매칭 실패. AI는 *같은 의미* 라고 압니다.

> 💡 한 마디로: **AI는 *글자*가 아니라 *뜻*을 봅니다.**

### 2.3 의미공간이 빛나는 사례 — 휴가 신청 사유 분류

> 시나리오: (주)멋진엔지니어링 관리부에 한 달 동안 들어온 휴가 신청 사유 50건. 사람마다 표기가 제각각.

| 휴가 신청 사유 (원본) | VLOOKUP | AI (의미공간) |
|---------------------|---------|--------------|
| `정기 건강 검진` | ❌ 매칭 실패 | ✅ **의료** |
| `치과 진료 (스케일링)` | ❌ 매칭 실패 | ✅ **의료** |
| `자녀 예방 접종 동행` | ❌ 매칭 실패 | ✅ **의료/가족** |
| `장인어른 칠순` | ❌ 매칭 실패 | ✅ **경조사** |
| `처제 결혼식` | ❌ 매칭 실패 | ✅ **경조사** |
| `이사로 인한 휴가` | ❌ 매칭 실패 | ✅ **개인사정** |
| `자녀 학교 행사` | ❌ 매칭 실패 | ✅ **가족** |
| `여름 휴가 (제주)` | ❌ 매칭 실패 | ✅ **휴양** |

→ AI는 *각 사유의 의미*를 읽고 **6개 범주**로 자동 분류 가능.
→ 엑셀로 하려면 *한 줄씩 사람이 직접 분류*해야 함 → 50건이면 한 시간.

**라이브 시연**: ChatGPT에 위 사유 8건만 보내고 *"이 휴가 사유들을 의미별로 묶어 표로 정리해줘"* 한 줄 → 결과 보여줌.

### 2.4 같은 항목, 맥락이 다르면 결과가 갈린다

> 시나리오: 같은 *"외부 미팅 참석"* 인데 *누구를 만나느냐*에 따라 비용 처리가 달라짐.

| 외부 미팅 사유 | 정답 처리 | 이유 |
|---------------|----------|------|
| 거래처 부장과 점심 | **접대비** | 외부 인사 |
| 협력업체 담당과 견적 협의 | **회의비** | 업무 협의 (식대 동반 X) |
| 본사 임원과 식사 | **복리후생비/회의비** | 사내 |
| 채용 후보자와 인터뷰 카페 | **채용 관련 경비** | 채용 절차 |
| `업무 미팅` (메모만) | *[확인필요]* | 적요만으론 분간 불가 |

→ AI는 **적요의 *맥락*** 을 읽고 처리 — *엑셀 수식으로는 절대 불가능한 일*.

### 2.5 의미공간의 함정 — AI는 *계산기*가 아닙니다 (3분)

> 교육생분이 자주 빠지는 함정 №1.

**❌ 잘못된 사용**
> "이 영수증 30건의 합계를 구해줘"
→ AI: *"대략 1,847,000원 정도 됩니다"* (실제 합계는 1,823,500원)
→ 교육생분: *"AI 거짓말이네!"* → 신뢰 상실.

**✅ 올바른 사용**
> "이 영수증 30건의 합계를 구하는 **엑셀 수식**을 만들어줘"
→ AI: *`=SUM(B2:B31)`*
→ 교육생분: 엑셀에 붙여넣기 → 정확한 합계 ✅

| 역할 | 누가 잘할까 |
|------|------------|
| **분류·판단·요약·다듬기** (의미공간 일) | 🤖 AI |
| **정확한 계산·합계·평균** | 📊 엑셀 / 코드 |

> 💡 **AI는 *연상 기계* 이지 *계산 기계*가 아닙니다.** 계산은 엑셀에게 양보하세요.

### 2.6 ⏱ 실습 ② — 휴가 사유 분류 (10분)

**자료**: 강사가 사전 배포한 `휴가신청_사유_50건.txt` (가공 데이터, 본문 1줄/건)

**진행**

1. 메모장으로 파일 열어서 전체 복사
2. ChatGPT 새 대화 → 다음 프롬프트 + 복사한 본문 붙여넣기

   ```
   [역할] 당신은 (주)멋진엔지니어링 관리부 인사 담당자입니다.
   [작업] 아래 휴가 사유 50건을 의미별로 분류해 주세요.
   [출력 형식] | # | 사유 | 분류 | 근거 |
   [데이터]
   (여기에 붙여넣기)
   ```

3. 결과를 5건 정도 *육안 검증*

**기록**

- 같은 의미인데 *표기가 다른* 사유 몇 건?
- 엑셀이라면 매칭 못 잡았을 사유 한 가지
- *분류가 애매한* 사유 한 가지

> ⏰ 시간 안 되면 30건만 해도 됨. 핵심은 *AI가 의미를 묶는 모습* 을 *육안으로* 보는 것.

## 2교시 (계속) — Part 3. 핵심 원리 3: 작업기억 (25분)

### 3.1 핵심 메시지 — *"AI는 매일 출근하는 신입사원"*

> AI에게 어제 *"우리 회사 회의록 양식은 이렇게 써 주세요"* 라고 알려줬더라도,
> 오늘 새 대화를 열면 — AI는 **모릅니다**.
> *어제의 모든 가르침은 사라졌습니다.*
> AI는 **매일 출근하는 신입사원** 같다고 생각하세요.

### 3.2 새 대화 = 완전 리셋

| 대화 ① (어제) | 대화 ② (오늘 새 채팅) |
|---|---|
| "당신은 회의록 담당자" 역할 부여 ✅ | 역할 없음 ❌ |
| 우리 회사 양식 5문단으로 설명 ✅ | 양식 정보 없음 ❌ |
| 발언자 표기 규칙 알려줌 ✅ | 규칙 없음 ❌ |
| → *완벽한 회의록* 결과 | → *위키백과처럼 어색한 회의록* 결과 |

> AI가 *바보가 된 게* 아닙니다.
> **그냥 *어제의 기억이 통째로 사라진* 것**.

### 3.3 교육생분이 자주 하는 착각 №1

> *"엇? 어제는 잘 됐는데 오늘은 왜 이상하지?"*
> *"AI가 일부러 그러나?"*
> *"내가 뭘 잘못 친 건가?"*

→ 아닙니다. **새 채팅을 여셨거나, 시간이 너무 지났거나, 다른 대화에서 작업한 것**입니다.
→ AI의 *기억력*은 본인 머리로 따라가지 마세요. **항상 0이라고 가정**하세요.

### 3.4 대응 전략 3가지 (5분)

**① 자주 쓰는 맥락을 *텍스트 파일로 저장*** ← *이게 곧 MD 프롬프트 파일* (3교시 본론)

**② Custom Instructions 활용** (ChatGPT *"나에 대해"* 설정)
> 본인 직무·회사·자주 쓰는 양식을 한 번 설정해두면 *모든 대화에 자동으로 반영*.
> 단, 이건 *내 계정* 안에서만. 동료에게 *공유 불가*.

**③ 재사용 가능한 프롬프트 *한 덩어리* 만들기**
> 회사 맥락 + 역할 + 출력 형식 + 예시 = 한 번에 묶어서 복사.
> 매번 새 채팅에서 *그 한 덩어리만* 붙여넣으면 됨.

> 💡 **다리 놓기 한 마디**: *"방금 ①번이 우리 교육의 핵심입니다. 3교시 시작하면 본인 손으로 한 번 만들어 보실 거예요."*

### 3.5 "클립보드 워크플로우" — 이번 교육의 *기본기*

> 도구가 ChatGPT든 Gemini든, 모델이 GPT-4든 GPT-5든 — **이 4단계는 살아남습니다**.

```
1️⃣  원본 데이터를 클립보드에 복사
       ↓
2️⃣  AI에게 *맥락과 함께* 붙여 요청
       ↓
3️⃣  결과를 다시 복사
       ↓
4️⃣  엑셀 / HWP / 메일창에 붙여넣기
```

> 이 단순한 4단계 — 그러나 교육생분의 70%는 *2단계*에서 *맥락*을 빼먹습니다. 오늘 그 습관을 고치는 게 목표.

### 3.6 ⏱ 실습 ③ — 신입사원 체험 (10분)

> 교육생분이 *직접 AI의 기억상실*을 체감하는 시간.

**진행**

1. 실습 ①에서 만든 *좋은 프롬프트*로 한 번 결과 받기 (이미 받았으면 그 결과 그대로)
2. **새 채팅 열기** (ChatGPT 왼쪽 *"새 대화"* 버튼)
3. 이번엔 **데이터만** 붙여넣고 *"정리해줘"* 한 줄
4. 두 결과를 *나란히* 비교

**기록**

| 칸 | 적기 |
|---|------|

## 3교시 — Part 4. ⭐ MD 프롬프트 파일 (30분)

> **이 회차의 *진짜 클라이맥스***. 1·2교시는 모두 이 시간을 위한 빌드업이었다. 슬라이드 8장.

### 4.1 핵심 메시지

> **MD 파일은 *AI에게 같은 일을 매번 시킬 수 있는 도구*입니다.**
> 한 번 만들어 두면 — *다음 달도, 그 다음 달도, 다음 분기도, 부서 이동을 해도* — **복사 한 번에 같은 결과**.

### 4.2 "텍스트는 텍스트가 아니다" — 같은 글, 3가지 표기 (5분)

같은 내용을 *세 가지 방식*으로 써 보면:

**(A) 메모장에 평범하게**

```
2026년 5월 행사 안내
일시 5월 30일 오후 2시
장소 본사 5층 대회의실
대상 전 임직원
준비물 사원증 명찰
```

→ 사람도 *읽기 힘들고*, AI도 *어디까지가 제목이고 어디부터가 내용인지* 모름.

**(B) 한글(HWP)로 표 그리고 셀 색깔 칠해서**

화려한 표·셀 색깔·중간 정렬·진한 글씨…
→ *사람*에게는 보기 좋아도, **AI에게 복사해 붙여넣는 순간 *서식은 모두 사라짐***.
→ 셀 안에 줄바꿈 들어가면 표 모양도 깨짐.

**(C) 마크다운으로**

```markdown
## 2026년 5월 행사 안내

| 항목 | 내용 |
|------|------|
| **일시** | 5월 30일 오후 2시 |
| **장소** | 본사 5층 대회의실 |
| **대상** | 전 임직원 |
| **준비물** | 사원증, 명찰 |
```

→ 사람도 보기 좋고, **AI에게 그대로 붙여 넣어도 *구조가 살아 있음***.

> 💡 마크다운은 **사람과 AI가 *모두 잘 읽는 텍스트***. 둘 다 만족하는 *공통어*.

### 4.3 Markdown 5문법 — 5분이면 끝 (5분)

> 교육생분 노트북에서 같이 띄워보게 하기: https://markdownlivepreview.com

| # | 문법 | 입력 | 결과 |
|---|------|------|------|
| 1 | **헤딩** | `# 큰 제목`  `## 중간 제목`  `### 작은 제목` | 글의 위계 |
| 2 | **목록** | `- 첫 항목`  `- 둘째 항목` | 글머리표 |
| 3 | **굵게** | `**중요**` | **중요** |
| 4 | **표** | `\| A \| B \|` 다음 줄 `\|---\|---\|` | 표 |
| 5 | **코드 블록** | ```` ``` ```` 로 위아래 감싸기 | 데이터 영역 분리 |

> **위 5개만 알면 AI 응답의 90%가 읽힙니다.**
> *나머지 10%는 평생 안 써도 됩니다.*

**라이브 시연 — markdownlivepreview.com**

1. 강사 화면에 사이트 띄움
2. 4.2-(C)의 마크다운 표를 *왼쪽*에 붙여넣기
3. *오른쪽에 표가 그려지는 모습* 함께 확인
4. **교육생분 노트북에서도 같이** 한 번 해보게 함 (이게 핵심 — *손이 움직여야 외워짐*)

> 💡 *"우리가 AI에게 받는 답이 *왜 자꾸 표·헤딩·굵은 글씨인지* 이제 아시겠죠? AI도 *마크다운으로 받으면 더 정확하게 일하기 때문*입니다."*

### 4.4 ⭐⭐ MD 프롬프트 파일 — 오늘의 *가장 중요한 15분*

> **여기서 교육생분이 *오늘 처음으로 자기 손으로* `.md` 파일을 만들어 봅니다.**

#### 4.4.1 MD 프롬프트 파일이 뭔가요?

교육생분의 PC 어딘가에 *조용히 잠들어 있다가*, 일이 생기면 *깨워서 쓰는* 텍스트 파일.

```
📂 내 PC \ 문서 \ 프롬프트 \
   ├── 회의록_정리.md
   ├── 거래처_메일_초안.md
   ├── 공문_다듬기.md
   ├── 안내문_작성.md
   ├── 휴가사유_분류.md
   └── 행사_안내문.md
```

**한 번 만들어 두면**:
- 다음 달 회의록 정리할 때 → `회의록_정리.md` 열어서 복사 → ChatGPT에 붙여넣기 → 이번 회의 메모만 끝에 추가 → 끝
- *매번 새로 프롬프트 쓰는 번거로움 0*

> ✨ 이것이 **본 1회차 교육의 핵심 산출물**입니다.

#### 4.4.2 MD 프롬프트 파일의 6개 영역 — *표준 템플릿*

> 이 6칸이 있는 메모장 파일 한 개 = 교육생분의 *업무 자동화 도구 1개*.

```markdown
# [작업명] 프롬프트

## [역할]
당신은 (주)멋진엔지니어링 관리부 ○○ 담당자입니다.

## [작업]
(한 줄로 — 무엇을 해달라는지)

## [판단 기준]
- (어떻게 결정할지 - 규칙 1)
- (규칙 2)
- (모호하면 [확인필요] 표기)

## [출력 형식]
(원하는 모양 - 표 / 목록 / 문단)

## [예시]
입력: (한 줄 샘플)
출력: (그에 대응하는 결과 샘플)

## [데이터]
(매번 여기에 새 데이터 붙여넣음)
```

| 영역 | 한 줄 설명 | 비유 |
|------|----------|------|
| **[역할]** | "당신은 누구다" | 신입사원에게 *명함* 주기 |
| **[작업]** | "이걸 해주세요" | 업무 *지시서* |
| **[판단 기준]** | "어떻게 결정할지" | *판단 매뉴얼* |
| **[출력 형식]** | "이런 모양으로 줘" | *결재 양식지* |
| **[예시]** | "이렇게 들어오면 이렇게 나가야" | *모범 답안* |
| **[데이터]** | **매번 갈리는 부분** | *오늘치 입력* |

> 💡 *"위 6칸을 한 번만 잘 채워두면, [데이터]만 갈아 끼우는 일이 평생 반복됩니다."*

#### 4.4.3 라이브 시연 — `회의록_정리.md` 만들어 보기

**시연 순서 (강사 노트, 약 5분)**

1. **메모장 열기** (Win 시작 → 메모장)
2. 4.4.2의 6칸 템플릿을 **그대로 복사 → 붙여넣기**
3. *회의록 정리용*으로 6칸 채우기 (강사가 빠르게 채우면서 *생각하는 과정*을 음성으로 같이 보여줌)
4. **파일 → 다른 이름으로 저장 → `회의록_정리.md`** (확장자가 `.txt`가 아닌 `.md`로 저장되도록 *모든 파일* 선택)
5. **ChatGPT 새 대화 열기 → 메모장에서 *전체 선택 → 복사* → ChatGPT에 *붙여넣기***
6. 마지막 `[데이터]` 아래에 *실제 회의 메모* 5줄 붙여넣기 → Enter
7. ✨ 결과 확인 → 교육생분에게 *"보세요, 사내 결재용 회의록 그대로죠?"*
8. **다른 회의 메모** 5줄로 같은 작업 → 같은 품질의 결과 ✅
9. **새 채팅 열어서 같은 파일** 다시 붙여넣기 → 또 같은 품질 ✅
10. 마무리 한 마디: *"이게 본 교육이 가는 길입니다."*

#### 4.4.4 교육생분이 빠지기 쉬운 *함정 3가지*

**함정 ①** ❌ *"매번 새로 프롬프트를 쓴다"*
> → *MD 파일로 한 번만* 쓰고 *복사 재사용*. 머리 안 굴려도 됨.

**함정 ②** ❌ *"AI 답변이 마음에 안 들면 *답변을 다시 시킨다*"*
> → 답변이 아니라 **MD의 [판단 기준]이나 [예시]를 *수정해서 저장***. 다음에 또 같은 문제 안 생김.

**함정 ③** ❌ *"내 비법 프롬프트는 내 머릿속에 있음"*
> → MD 파일로 만들어 두면 **부서원에게 *공유 가능***. 후임자 인수인계도 *파일 전송* 한 번.

#### 4.4.5 MD 프롬프트 파일 vs 평범한 메모

> **이 한 장의 표가 1회차에서 교육생분이 가져가는 *가장 중요한 인사이트***.

| 비교 항목 | 평범한 메모 | MD 프롬프트 파일 |
|---------|-----------|---------------|
| 다음 달에 다시 쓸 수 있나? | "어떻게 썼더라..." | 파일 열면 끝 |
| 동료에게 공유 가능? | 말로 설명 (10분) | 파일 전송 (3초) |
| AI 결과 품질 | *그때그때 다름* | **매번 같은 수준** |
| 개선 가능? | 머릿속에서 어림 | 파일을 *고쳐서 저장* |
| 휴가·이직 시 인수인계 | "본인이 잘 해보세요" | "이 폴더 통째로 드릴게요" |

## 3교시 (계속) — Part 5. 인터뷰 프롬프트 (10분)

### 5.1 핵심 메시지 — *"AI를 *인터뷰어*로 바꿔 보자"*

> 지금까지의 MD 프롬프트는 모두 **교육생분 → AI** 방향이었습니다.
> *"이거 해주세요"* 하면 AI가 결과를 줌.
>
> **인터뷰 프롬프트는 *방향이 반대*** 입니다.
> *"당신이 인터뷰어다, 나에게 질문해라"* — AI가 교육생분에게 **질문**하기 시작합니다.
> 교육생분은 답만 하다 보면 **본인도 몰랐던 본인 업무 지도**가 완성됩니다.

### 5.2 두 가지 MD 프롬프트 — 한눈에 비교

| 일반 MD 프롬프트 | **인터뷰 MD 프롬프트** |
|----------------|---------------------|
| 교육생분 → 데이터 → AI → 결과 | AI → 질문 → 교육생분 → 답 → AI → 다음 질문 → … |
| 1턴 (한 방에) | **여러 턴 대화** |
| 데이터가 *이미 있을 때* 적합 | 데이터가 *내 머릿속에만 있을 때* 적합 |
| 예: 회의록 정리, 휴가 분류 | 예: 본인 업무 정리, 도구 사양 정리, 신입사원 OJT |
| 회의록 담당자, 분류기, 작성기 | **신문기자, 컨설턴트, 코치** |

> 🎯 **인터뷰 프롬프트가 빛나는 순간**: *"내가 뭘 모르는지 모를 때"*, *"머릿속에 있는데 글로 못 쓸 때"*.

### 5.3 라이브 시연 — `homework-interviewer.md` (5분)

> 본 1회차 과제용 인터뷰 프롬프트. 강사가 *직접* 시연.

**시연 순서**

1. `homework-interviewer.md` 파일을 *메모장으로* 열어 강사 화면에 띄움
2. 교육생분에게 한 마디: *"이건 그냥 *텍스트 파일* 입니다. *마법의 부적*이 아닙니다. 안에는 *AI에게 인터뷰어가 되어달라*는 *부탁의 글*이 마크다운으로 쓰여 있을 뿐."*
3. ChatGPT 새 대화 → 전체 복사 → 붙여넣기 → Enter
4. AI의 **첫 질문**이 나옴: *"안녕하세요. 먼저 어떤 부서에서 어떤 역할을 맡고 계신지 말씀해 주시겠어요?"*
5. 강사가 *본인 역할*로 한 줄 답변 → AI가 *후속 질문*
6. 1~2턴 더 진행 → *AI가 잊었던 것까지 파고드는 모습* 보여줌
7. 마무리 한 마디: *"보셨죠? 오늘 저녁 본인 업무로 *이걸 그대로 하시면* 됩니다. 그게 과제."*

### 5.4 인터뷰 프롬프트는 *전 회차의 골격*

| 회차 | 어떤 인터뷰 프롬프트? | 결과물 |
|------|--------------------|-------|
| **1회차** (오늘 과제) | `homework-interviewer.md` | **업무 맥락 정의서** |
| 2회차 | 업무 흐름도용 (Mermaid 함께) | 업무 흐름도 + 도구 컨셉 |
| 3회차 | `micro-tool-interviewer.md` | HTML 도구 사양서 |

> 💡 *"본 교육 9시간이 끝나면, 본인의 *프롬프트 폴더*에는 *일반 MD 프롬프트* 여러 개 + *인터뷰 MD 프롬프트* 1~2개가 쌓여 있을 겁니다. 이게 *진짜 자산* 입니다."*

## 3교시 (계속) — Part 6. 자동화 후보 발굴 (10분)

### 6.1 자동화에 *적합한* 작업 — 4가지 조건

> 오늘 배운 **3-핵심 원리 + MD 프롬프트** 잣대로 본인 업무를 *훑어보는* 시간.

**✅ MD 프롬프트가 빛나는 작업**

- 매주·매월 **반복** 발생
- 입력과 출력 *형식이 정형화* 가능
- **판단보다 *처리*** 가 많은 작업 (계산이 아니라 *분류·요약·다듬기*)
- 한 번 만들면 **나만 써도** 가치 있음

**❌ 적합하지 않은 작업**

- 1년에 1~2번뿐인 작업 (만들 시간이 더 듦)
- 매번 결과물 양식이 *완전히 달라지는* 작업
- **결정 권한·책임**이 필요한 작업 (인사 평가, 계약 결정 등)
- *정확한 숫자 계산* 만 하는 작업 (엑셀이 더 빠름)

### 6.2 관리부서 자동화 사례 — 영감 얻기

> *"이 표를 보면서 본인 업무에서 *비슷한 게* 떠오르는지 보세요."*

| 부서 | 후보 예시 | MD 파일 이름 (가칭) | 한 번 만들면 시간 절감 |
|------|---------|------------------|-------------------|
| 총무 | 회의록 정리 | `회의록_정리.md` | 30분 → 3분 |
| 인사 | 휴가 사유 분류·집계 | `휴가사유_분류.md` | 1시간 → 5분 |
| 인사 | 채용 공고 문구 다듬기 | `채용공고_다듬기.md` | 2시간 → 15분 |
| 구매 | 거래처 답신 메일 초안 | `거래처_메일_초안.md` | 20분 → 2분 |
| 자산 | 자산 카드 분류 | `자산_분류.md` | 1시간 → 10분 |
| 총무 | 안내 문자 작성 | `안내문자_작성.md` | 10분 → 1분 |
| 일반 | 보고서 초안 다듬기 | `보고서_다듬기.md` | 1시간 → 15분 |
| 일반 | 공문 톤앤매너 다듬기 | `공문_다듬기.md` | 30분 → 5분 |

> 💡 *"매월 *같은 일* 을 *반복* 하고 있다 — 그게 *후보의 신호* 입니다."*

### 6.3 ⏱ 실습 ④ — 본인 업무 후보 3개 (10분)

> **1회차의 *진짜 결과물***. 2회차 도구 컨셉 확정의 *재료*.

**워크시트**

| # | 후보 (작업명) | 입력 → 출력 | 빈도 | 1회 소요 | MD 파일 이름 (가칭) |
|---|-------------|------------|------|---------|--------------------|
| 1 | (예: 회의록 정리) | 회의 메모 → 회의록 | 주 1회 | 30분 | `회의록_정리.md` |
| 2 |  |  |  |  | `_____.md` |
| 3 |  |  |  |  | `_____.md` |

**작성 팁**

- 떠올리는 기준: *"내가 매주 / 매월 *같은 패턴* 으로 하는 일은 뭐지?"*
- 너무 거창하게 X. *"메일 초안 한 줄 잡아주는 것"* 정도도 훌륭한 후보.
- *파일 이름*까지 적어두면 — 다음 회차에 *그 이름 그대로* 만들어집니다.

> 🚀 시간이 모자라도 **3개 칸은 다 채우세요**. 다음 달 본인의 *업무 자동화 도구 후보 리스트*가 됩니다.

### Slidev 작성 시 주의

- **한 슬라이드 = 한 메시지** 원칙. 글머리표 5개 이내, 표는 행 6개 이내
- 라이브 시연 슬라이드는 본문 *최소*, 강사 노트(`<!-- -->`)에 시연 스크립트
- `<v-clicks>` 로 글머리표 *한 줄씩 등장* — 발표자 페이스 잡기
- `layout: two-cols` 로 실습 슬라이드 좌우 분할 (왼쪽 진행 / 오른쪽 기록 워크시트)
- **교시 구분 슬라이드 3장** (`layout: section`) — 교육생분이 *지금 어디쯤* 인지 즉시 알 수 있게
- **휴식 슬라이드 2장** — 큰 글씨 ☕ + 다음 교시 *주제 1줄*
- **이모지·비유 적극 사용** — 다만 *진행 도중* 교육생분이 *공감*할 수준만 (남발 금지)
- 회사명은 슬라이드 어디에도 **"유신"** 노출 ❌ → **(주)멋진엔지니어링** 통일
