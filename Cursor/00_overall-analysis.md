새 1회차 시나리오의 핵심 — **"회의록 정리"가 본 회차의 *대표 시나리오***:

→ **회의록 시나리오를 처음부터 끝까지 한 가닥으로 끌고 가는 *서사*** 가 위원의 기억에 박힌다.

## 3. 새 1회차 콘텐츠 설계 (Mermaid 직전까지)

### 3.1 1회차 도달 상태 — "이걸 할 수 있게 만든다"

본 교육의 **사용자 모델** (대상자가 도달해야 하는 상태):

- **ChatGPT / Gemini 대화창만 사용** — 코딩·HTML 도구 제작은 본 1회차 범위 외
- **MD 파일로 프롬프트를 작성·저장·재사용** 하는 능력
- 본인의 **반복되는 사내 업무**를 *MD 프롬프트 + ChatGPT 대화* 조합으로 매번 해결

1회차 종료 시점에 위원의 도달 상태 4가지:

1. AI가 왜 그런 답을 내놓는지 **3원리(자동완성·의미공간·작업기억)** 로 설명 가능
2. AI에게 줄 글을 **마크다운으로 정리** 가능 (지시문/데이터 영역 분리)
3. 본인 업무용 **MD 프롬프트 파일 1개의 뼈대**를 잡을 수 있음 ← **본 교육의 핵심 산출물**
4. 본인 업무에서 자동화할 **후보 3개**를 골라낼 수 있음 (2회차 도구 컨셉 재료)

### 3.2 시간 배분 — 50분 × 3교시 + 휴식 10분 × 3 = 180분

| 교시 | 시간대 | 큰 주제 | 키 메시지 |

| 휴식 (10) | 0:50 ~ 1:00 | — | — |

| 휴식 (10) | 1:50 ~ 2:00 | — | — |

> **실습 총량**: ① 15분 + ② 10분 + ③ 10분 + ④ 10분 = **45분 (전체의 25%)**.
> 2회차에서 50%, 3회차에서 70% 이상으로 점진 확대.

> ⭐ 3교시 중반의 **"MD 프롬프트 파일" 15분이 1회차의 진짜 클라이맥스**. 1·2교시는 모두 *이 시간을 위한 빌드업*이다.

> 기존 `ppt/slides.md`는 약 30 슬라이드로 위 흐름의 약 70%를 커버. **Part 0 교육과정 안내 + MD 프롬프트 파일 확장 + 인터뷰 프롬프트 슬라이드 약 14장 신규 추가**로 1회차 자료(약 44장)를 완성한다.

## 4. 실습·과제 데이터 흐름

```
[1회차 자체 산출물]
  ├─ 실습 ① 본인 업무 한 작업 → 나쁜/좋은 프롬프트 비교 결과
  ├─ 실습 ② 휴가 신청 사유 50건 → 의미별 분류 결과
  ├─ 실습 ③ 신입사원 체험 (새 채팅 = 리셋) 결과
  ├─ 실습 ④ 자동화 후보 3개 (워크시트, MD 파일 이름 포함)
  └─ [회차 간 과제]
       ① 업무 맥락 정의서 (homework-interviewer)
       ② 자동화 후보 3개 워크시트
       ③ (선택) MD 프롬프트 파일 시범 작성 1개
       ↓
[2회차로 인계]  업무 흐름도(Mermaid) 작성의 재료 + 도구 컨셉 확정의 재료
```

### 4.1 1회차에서 위원이 다운로드해야 하는 파일

## 5. Slidev 빌드 / 배포 흐름 (참고)

`Slidev 셋업 및 배포 가이드.md` 핵심을 도식화하면:

```
1. 처음 한 번 : Node.js 22.x.x 설치 + PowerShell ExecutionPolicy 풀기
2. 회차별 폴더 : npm install
3. 작성 중    : npm run dev → http://localhost:3030
4. 빔 시연    : f(전체화면) + . (네비바 숨김) + d(펜) + g(슬라이드 점프)
5. 배포       : git push → GitHub Actions가 npm run build → Pages
6. 위원 배포  : npm run export → slides-export.pdf
```

→ 우리는 `ppt/` 폴더가 이미 1회차 슬라이드 작업 폴더로 셋업되어 있으므로, **개편 후 같은 폴더에서 `npm run dev`로 검증 → push로 자동 배포** 가능.

### 5.1 GitHub Pages + Gist 배포 옵션

- **GitHub Pages**: 회차별 단일 URL 생성. `https://{사용자}.github.io/{레포}/`
- **Gist 배포**: Slidev 자체는 정적 빌드물(HTML+JS+CSS)을 만들기 때문에 Gist만으로는 한계가 있음. → **Gist에는 `slides.md` 소스만 공유**하고, 실제 슬라이드 시연·공유는 Pages URL을 권장.
- (대안) GitHub Pages 대신 Vercel/Netlify에 무료 배포도 가능 — Pages가 셋업 가장 단순.
